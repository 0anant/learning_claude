How an AI API Request Really Works

The complete journey from a user typing a question to receiving an AI-generated response — every step explained.

Client
Server
Claude API
1

Client Your Server

The user submits a message (e.g. "What is quantum computing?") from the web or mobile app. This request goes to your server, never directly to the API.

2

Your Server Anthropic API

Your server attaches the secret API key and forwards the request to the Anthropic API. The request must include: model, messages, and max_tokens.

3

Claude Processes the Request

The model processes the input through four internal stages: tokenization, embedding, contextualization, and generation — one token at a time.

4

Anthropic API Your Server

The API returns a structured response containing the generated message, token usage, and the stop reason back to your server.

5

Your Server Client

Your server relays the response to the client. The user sees Claude's answer on their screen. Cycle complete.

Why You Need a Server

Never call the Anthropic API directly from client-side code. Here's why:

API requests require a secret API key for authentication
Exposing your key in client code is a critical security vulnerability
Anyone can extract the key from the browser and make unauthorized requests
Your server securely stores the key and acts as a trusted proxy

Required Request Fields

Every API request must include these four fields:

api_key Authenticates the request — identifies who's calling
model Which model to use (e.g. claude-sonnet-4-20250514)
messages The conversation history — a list of user/assistant messages
max_tokens Upper limit on how many tokens the model can generate

Inside Claude's Processing

When Claude receives your request, it goes through four stages:

Tokenization

The input text is split into tokens — words, subwords, or symbols that the model can process.

Embedding

Each token is converted into a high-dimensional vector of numbers that captures its semantic meaning.

Contextualization

Embeddings are adjusted based on surrounding context. For example, "quantum" in physics vs. finance gets different representations.

Generation

The output layer computes probabilities over the vocabulary and selects the next token. This repeats one token at a time until a stop condition is met.

When Claude Stops Generating

After each token, Claude checks: should I keep going? Generation stops when:

The max_tokens limit is reached
An end-of-sequence marker is generated
A predefined stop phrase is encountered

The API Response

Once generation completes, the API returns a structured JSON response with three key fields:

message
The generated text content from Claude
usage
Count of input and output tokens consumed
stop_reason
Why generation ended (max_tokens, end_turn, stop_phrase)
